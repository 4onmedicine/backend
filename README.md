## ⚠️ commit 컨벤션

```
{태그}: {클래스 이름} {커밋 메시지}
```

- 💡 예시: `[Feat] 회원 가입 기능 구현`
- 커밋 내용은 명사로 끝나며 마침표를 사용하지 않는다.

### 태그

- 이모지는 선택에 따라 활용한다.

| 태그       | 설명                      |
|:---------|:------------------------|
| Feat     | 새로운 기능 구현               |
| Fix      | 버그, 오류 수정                   |
| Docs     | README와 같은 문서 수정        |
| Test     | 테스트 코드 추가 및 업데이트        |
| Refactor | 코드 리팩토링                 |
| Comment  | 주석 추가(코드 변경 X) 혹은 오타 수정 |
| Merge    | 다른 브랜치를 merge 할 때 사용                   |
| Add   | Feat 이외의 부수적인 코드 추가, 라이브러리 추가, 새로운 파일 생성 시        |
| Rename   | 파일 이름 변경        |
| Move   | 프로젝트 내 파일이나 코드의 이동        |
